# 智科分工说明

智科方向负责智能家居系统中的数据、识别、界面和控制闭环。

## 我们负责什么

1. 数据管理
   - 保存授权人员信息。
   - 保存温度、门窗、灯光、风扇状态。
   - 保存摄像头图片和识别结果。
   - 保存远程控制命令。

2. 智能识别
   - 使用摄像头图片进行目标识别。
   - 演示识别无人机、灭火器、灯泡三类。
   - 实现 2 个授权人员和 1 个未授权人员的人脸门禁逻辑。
   - 人脸识别模块负责授权判别；无人机、灭火器和灯泡不会触发门禁开门。

3. GUI 展示
   - 展示当前智能家居状态。
   - 展示图片识别结果。
   - 展示人脸门禁结果。
   - 展示历史数据。

4. 远程控制
   - 通过 GUI 发送开门、关门、开灯、关灯、调亮度、开风扇、关风扇命令。
   - 硬件组通过接口拉取命令并执行。

5. 创新功能
   - 当摄像头检测到灯泡图片时，自动生成开灯命令，但不会开门。
   - 当温度过高时，自动生成开风扇命令。

## 和其他专业的接口

电管/电子/物联网同学负责硬件模型、传感器和执行器。

智科系统提供 HTTP 接口：

- 硬件上传传感器数据到智科后端。
- 硬件上传摄像头图片到智科后端。
- 硬件从智科后端拉取控制命令。
- GUI 从智科后端读取所有数据并展示。

## 阶段 1 目标

阶段 1 不要求完整硬件联调，但智科组件必须能独立运行：

- 后端能启动。
- 数据库能保存记录。
- GUI 能打开。
- 图片上传后能产生识别结果。
- 授权人员能触发开门命令。
- 未授权人员会拒绝进入。
- 历史数据能查询。

## 阶段 2 目标

阶段 2 与硬件模型集成：

- 接入真实温度传感器。
- 接入真实门窗状态。
- 接入真实灯光和风扇控制。
- 接入真实摄像头图片。
- 优化目标识别模型和展示效果。
